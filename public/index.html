cloudInformación del Backend

Bienvenido a la documentación del backend para la aplicación de cursos. Este backend está construido en **Node.js** y **Express**, y proporciona una API para gestionar usuarios, cursos y progreso de estudiantes.

link Enlace al Frontend

El frontend de esta aplicación está desarrollado en React, utilizando Vite y Material UI para proporcionar una experiencia de usuario moderna y responsiva.

integration_instructions Detalles de Integración
  • Autenticación

    El backend proporciona una API de autenticación para el registro e inicio de sesión de usuarios. Los tokens JWT se utilizan para mantener sesiones seguras.

  • Gestión de Cursos

    Los endpoints de cursos permiten al frontend obtener la lista de cursos disponibles, así como filtrar y paginar los resultados. Los usuarios autenticados también pueden asignarse cursos.

  • Progreso de Estudiantes

    Cada usuario puede llevar un registro de su progreso en los cursos asignados. Los datos se sincronizan entre el frontend y el backend a través de la API de progreso.

api API Endpoints
Endpoint Descripción Método
/auth/register Registrar un nuevo usuario POST
/auth/login Iniciar sesión y obtener token JWT POST

dashboard Información sobre nuestros Tableros de Trello

Contamos con dos tableros de Trello dedicados a diferentes aspectos de nuestro proyecto:

Navegación con Filtros

En este tablero, gestionamos el desarrollo de un sistema de navegación con filtros. Todas las tareas en el backlog de sprint han sido completadas, moviéndose a la columna de "Hecho" y listas para producción. Cada tarea está vinculada a su historia de usuario, mostrando los criterios de aceptación correspondientes, simulando un despliegue en el que los servicios ya se encuentran en línea.

Visualización de Progresos de Cursos

Este tablero organiza el progreso de los cursos, con una priorización de cinco niveles. En el backlog del primer sprint, se seleccionaron tareas de nivel 1. Todas las tareas están conectadas a sus respectivas historias de usuario para asegurar que cumplan con las necesidades del proyecto.

Priorizaciones y Recursos

Se han utilizado etiquetas de priorización y se han vinculado branches de GitHub en algunos casos para facilitar la conexión con los repositorios de backend y frontend. Además, los siguientes recursos están enlazados en las tareas del tablero:

  • Branches en GitHub
  • Miro para diagramas de flujo
  • Figma para diseño de interfaces
  • Consola de MongoDB (acceso para usuarios autorizados)
  • Render para despliegues (acceso para usuarios autorizados)
